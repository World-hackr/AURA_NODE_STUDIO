# Mindmap Use

Use a mindmap only for hierarchy-first thinking.

Good use cases:

- host menu grouping
- setup flow grouping
- expert settings grouping
- feature breakdown

Do not use mindmap files for final exact logic.

## Best Way To Use

1. Open `host_menu_tree.km` in this folder first.
2. If that opens correctly, use the same `.km` format for future maps.
3. Save new files inside this folder with a clear name like:
   - `host_menu_tree`
   - `node_setup_tree`
   - `expert_settings_tree`

## Recommendation

Use one mindmap extension as the main one.

If multiple mindmap extensions are installed, do not use all of them daily.
Pick one as the primary editor so your files stay consistent.

## Recommended Primary Extension

Use `oorzc.mind-map` as the main `.km` editor.

Why:

- it directly supports `.km` and `.xmind`
- its README clearly matches the file workflow
- it is less specialized than the enterprise/test-case focused variants

## Conflict Note

The `souche` and `eighthundreds` mindmap variants are closely related and can conflict.
Do not rely on both as daily editors for the same files.
